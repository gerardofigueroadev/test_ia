Powered by Claude Opus 4.6 · GitHub Actions · Trello

El Problema

La deuda técnica y las vulnerabilidades pasan al código de producción

👁️

Code reviews manuales fallan

Los reviewers humanos pasan por alto credenciales expuestas, inyecciones SQL y algoritmos inseguros bajo presión de tiempo.

📋

La deuda técnica no se registra

Aunque se detecta un problema, rara vez se crea un ticket formal. Queda en comentarios del PR que nadie revisa después.

⏱️

Tiempo de respuesta tardío

Los problemas se detectan semanas o meses después del merge, cuando el costo de corregirlos es mucho mayor.

📉

Sin trazabilidad

No hay visibilidad de cuánta deuda técnica acumula el proyecto ni qué tan crítica es cada vulnerabilidad.

La Solución

Dos automatizaciones que cubren todo el ciclo de vida de un cambio

🔍

Revisión en Pull Request

Cada PR recibe automáticamente un code review de Claude con criterios de seguridad, buenas prácticas y calidad.

→
🔀

Merge a Main

Si el código llegó a main con problemas sin resolver, se detectan y se crea un ticket en Trello automáticamente.

→
🎫

Ticket en Trello

La deuda técnica queda documentada, priorizada por severidad y asignada al backlog del equipo.

¿Qué Detecta?

Criterios de análisis aplicados en cada revisión automática

Credenciales hardcodeadas — API keys, passwords, tokens, SSH keys en el código fuente

Vulnerabilidades OWASP — SQL Injection, Path Traversal, Command Injection, XSS

Algoritmos inseguros — MD5/SHA1 para contraseñas, JWT con clave débil

Exposición de datos sensibles — Contraseñas, IPs internas o secretos en respuestas HTTP

Problemas de lógica — Backdoors, tokens predecibles, comparaciones inseguras

Buenas prácticas — Ausencia de validación, falta de autenticación, logs con datos sensibles

Optimización de Costos

Cómo reducir el gasto en Claude API de ~$0.05 a ~$0.005 por revisión

80%

Modelo por tamaño de diff

Diffs pequeños (<100 líneas) usan Haiku (10x más barato). Solo escalar a Opus en diffs grandes o archivos críticos.

30%

Prompt Caching

El system prompt es idéntico en cada llamada. Con cache_control Anthropic lo cachea 5 min y cobra solo el 10% en llamadas seguidas.

40%

Filtrar archivos irrelevantes

Excluir *.lock, *.yml, *.md, imágenes y dist/ del diff antes de enviarlo. No aportan al análisis de seguridad.

—

Umbral mínimo de líneas

Si el diff tiene menos de 10 líneas (typo, comentario) se omite la llamada por completo.

—

Desactivar Thinking en diffs simples

adaptive thinking agrega tokens internos de razonamiento. Para diffs triviales se puede omitir sin perder calidad.
